# Real-time Weather Monitor & Analyzer

โปรเจกต์นี้สาธิตการจัดการการทำงานแบบคู่ขนาน (Concurrency & Parallelism) ในภาษา Python ผ่านระบบดึงข้อมูลอุณหภูมิแบบเรียลไทม์ โดยแบ่งหน้าที่ตามความเหมาะสมดังนี้:

## 1. Asynchronous I/O (`asyncio`)
- **หน้าที่ในโค้ด:** ดึงข้อมูลอุณหภูมิจาก API (`wttr.in`) ในหลายๆ จังหวัดพร้อมกัน
- **ทำไมต้องใช้:** การดึงข้อมูลผ่านเครือข่ายเป็นงานประเภท **Network I/O-Bound** การใช้ `asyncio` ช่วยให้โปรแกรมส่งคำขอไปดึงข้อมูลทุกจังหวัดได้พร้อมกันโดยไม่ต้องรอ (Non-blocking) ทำให้ดึงข้อมูลได้รวดเร็ว

## 2. Threading (`threading`)
- **หน้าที่ในโค้ด:** สร้างระบบ System Monitor เบื้องหลัง เพื่อแสดงสถานะ Loading (Spinner) ออกทางหน้าจอ
- **ทำไมต้องใช้:** เป็นงานประเภท **I/O-Bound** การรันบน Thread ช่วยให้โปรแกรมหลักสามารถแสดงผลสถานะอัปเดตไปได้เรื่อยๆ โดยไม่ขัดจังหวะการดึงข้อมูลและพร้อมปิดตัวเองลงเมื่อโปรแกรมหลักทำงานเสร็จ

## 3. Multiprocessing (`ProcessPoolExecutor`)
- **หน้าที่ในโค้ด:** นำข้อมูลอุณหภูมิจริงที่ดึงมาได้ ไปเข้าสมการทางคณิตศาสตร์จำลองเพื่อวิเคราะห์สภาพอากาศ (วนลูป 5 ล้านรอบ)
- **ทำไมต้องใช้:** การคำนวณที่ซับซ้อนจำนวนมากเป็นงานประเภท **CPU-Bound** การใช้ Process Pool จะสั่งให้ Python แตกงานนี้ไปรันบน CPU Core อื่นๆ เพื่อให้ได้ความเร็วสูงสุดและข้ามขีดจำกัดเรื่อง GIL ของ Python
